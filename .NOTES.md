* Edge Project is the name of the project, we’ll be using NextJS framework and Supabase for database and auth, Posthog for analytics, Vercel for hosting. The website domain will be edgeproject.com
So basically edgeproject is an PWA website, students/users can order food from particular shops/canteen available and pay from their mobile phone and receive the food parcel in an instance instead of waiting in the queue asking for an order. There will be number of shops available in the edgeproject, we are only building the system to order food online so the students/users can collect them at the food vendors which avoid the line and the food shop have time to prepare the food and for the number of customers. The users can order the food by going to the specific shop like edgeproject.com/shop/rocksweats or just discover the menus at edgeproject.com where foods/beverage/sweets/drinks/juice etc.. can be browsed from all the available shops (user can add the shops which are to be displayed - the user can filter out).

The initial workflow for the v1.0 is like this, the user/student go to edgeproject.com then browse through the menu of multiple shops with their food are presented (the user can add to fav foods, personalize/filter out the shops at the browse page) then the user add the food to the cart or click order, then the user will be asked for more details like to add notes for the food, quantity, dining or parcel, estimated time of pickup, add more items etc… and then finally when the user want to proceed with the selection then the system will redirect the user for payment of the food and verify if the payment is being made - if yes then the user will get the receipt and which include details about the food, and everything. This receipt is the key to take the food parcel from the shop, the user need to tell the code (another short auto generated code for the ease of use to verify) to the shop counter to pick up after the food parcel has prepared.
