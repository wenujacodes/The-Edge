### Backend & Order Flow Architecture

- **Multi-Shop Sequential Checkout:**
    - If a cart contains items from multiple shops, payments are processed sequentially (one shop at a time).
    - "Pay Securely" triggers a guided flow: Pay Shop A -> Success -> Pay Shop B.
    - If a user cancels payment for one shop, those specific items remain in the cart with all notes/quantities preserved.
    - Separate receipts (with unique codes) are generated for each shop.

- **Daily Resetting 4-Digit Codes:**
    - Orders are assigned a 4-digit code (e.g., 0001) that resets every day at 00:00.
    - User gives this code to the shopkeeper for fast daily pickup.

- **Scheduling & ASAP Logic:**
    - Scheduling is handled directly on the Cart page (no separate checkout page).
    - Individual food items can be scheduled (Dine-in/Takeaway buttons) or set to ASAP.
    - Shops can configure specific time slots (e.g., 9 AM, 10 AM, 2:30 PM) per food item.
    - **Priority Engine:** Scheduled orders take priority in the vendor queue. Sorting is based on "Target Time" (Scheduled Time vs. Payment Time for ASAP).

- **Simplified Vendor Dashboard:**
    - **No "Preparing" Pane:** Workflow is reduced to `[Make Ready]` -> `[Complete/Served]`.
    - `Make Ready` notifies the user. `Complete` moves the order to history.
    - **Undo Action:** Vendors can revert "Make Ready" if clicked by mistake.
    - **Smart Search:** Typing "1" finds "0001".
    - **Receipt Preview:** A dedicated pane on the right shows the exact customer receipt when an order is selected.
    - **Audio Alerts:** A "Ping" sound plays for every new incoming paid order.
    - **Search & Filters:** Default view is Today, with filters for This Week and This Month.

- **Cart & Favorites Sync:**
    - Data is stored in local storage and synced with the Supabase backend if the user is logged in.
    - Real-time synchronization across devices.

### Critical Backend & UI Friction Points

- **Midnight Code Reset Gap**
    - **Problem:** Orders placed at 11:55 PM (e.g., Code 8500) might "disappear" or conflict when the sequence resets to 0001 at 12:00 AM.
    - **Fix:** Vendor panel filters by "Active" status (not just today's date). An order keeps its assigned code until marked `Completed`, regardless of the midnight reset.

- **Sequential Payment Failure Recovery**
    - **Problem:** If a user pays for Shop A but the payment for Shop B fails, the cart might become desynced.
    - **Fix:** Atomic shop-level processing. Successful payments move items to `Orders` and clear them from `Cart`. Failed payments keep items in `Cart` with a "Payment Failed - Try Again" badge.

- **Real-Time Vendor Pulse**
    - **Problem:** Vendors may miss orders if the screen doesn't update or if they aren't looking.
    - **Fix:** Use Supabase Realtime for instant order "pop-in." Trigger a distinct "Ping" audio alert for new paid orders.

- **"Air Ticket" Printability**
    - **Problem:** Vendors often need physical tickets for the kitchen.
    - **Fix:** Add CSS `@media print` styles optimized for 58mm/80mm thermal printers. Include high-contrast "PAID" or "READY" watermarks.

- **Smart Search Optimization**
    - **Problem:** Instant search on every keystroke can lag or overwhelm the database.
    - **Fix:** Implement 300ms debouncing on the search bar. Use database indexing on `daily_code` and `reference_number` for sub-second retrieval.
